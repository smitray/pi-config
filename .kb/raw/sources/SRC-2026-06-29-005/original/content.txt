# KB Cross-Extension Reuse Opportunities

## Already Done
- _shared/result.ts: KB uses ok()/err() for all tool results
- guardrails/api.ts: KB calls registerRules() to protect .kb/raw/ and .kb/meta/
- web-access/lib/markdown.ts: KB uses approxTokens() and splitIntoChunks() for source capture

## Code Reduction Opportunities

### 1. Extract fetchWithRetry to _shared/http.ts (save ~90 lines in KB)
KB's lib/models.ts has 279 lines, of which ~90 are fetchWithRetry (exponential backoff on 429/5xx).
web-access has similar retry logic in lib/http.ts. Both could use one shared implementation.
Current: 279 lines → After: ~180 lines in models.ts, +30 in _shared/http.ts

### 2. Replace inline KNOWLEDGE_KEYWORDS pattern
KB's before_agent_start hook checks ~50 hardcoded keywords to gate auto-recall.
Could use a configurable pattern or delegate to a skill contract instead.

## Feature Additions via Existing Extensions

### URL Routing in kb_capture (+30 lines)
kb_capture currently only takes text/file. Add url parameter with internal routing:
- GitHub URL → call gh-repo-view / gh api → capture result
- Docs URL → call web-fetch (single) or web-fetch-docs (recursive)
- YouTube URL → call media-transcribe
Collapses 3-step manual workflow into 1 step.

### Confirm Guardrails for Wiki Ops (+5 lines)
KB only uses 'block' action. Add 'confirm' for destructive wiki edits:
- Editing wiki files outside kb_ensure_page → confirm (may break backlinks)
Already have the API via guardrails/api.ts.
